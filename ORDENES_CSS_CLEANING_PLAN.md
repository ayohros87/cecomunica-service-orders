# Ordenes CSS Cleaning Plan (Aggressive)

## Goal
Reduce `public/css/ordenes-index.css` by removing unused/duplicate styles, consolidating components, and reorganizing into a single, coherent file while keeping all current UI/UX intact.

## Scope
- **Target file**: `public/css/ordenes-index.css`
- **Reference files**:
  - `public/ordenes/index.html`
  - `public/js/ordenes-index.js`
  - `public/css/ceco-ui.css` (verify shared tokens/classes)

## Phase 1 — Inventory & Usage Mapping
1. Extract all selectors from `ordenes-index.css`.
2. Map selectors to actual usage in `index.html` and DOM generated by `ordenes-index.js`.
3. Mark selectors as:
   - **Used** (keep)
   - **Generated** (created by JS at runtime)
   - **Unknown/Legacy** (not found)
4. Identify duplication:
   - repeated keyframes (`pulse`, etc.)
   - repeated utility classes (e.g., clamp)
   - overlapping component styles

## Phase 2 — Aggressive Removal (Safe Set)
Remove only when **both** conditions are true:
- not referenced in HTML/JS
- not required by shared global styles in `ceco-ui.css`

Apply removals in batches:
- **Batch A**: unused legacy utilities
- **Batch B**: duplicated keyframes
- **Batch C**: unused component blocks
- **Batch D**: unused mobile-only blocks

## Phase 3 — Consolidation
Consolidate into a single file with clear sections:

1. **Base / Tokens** (root vars, body, typography)
2. **Layout** (topbar, containers, grids)
3. **Tables** (orders table, expanded row)
4. **Buttons / Inputs** (shared controls)
5. **Components**
   - resumen
   - overflow menus
   - accessories
   - intervention
   - cards (mobile)
6. **Modals**
   - asignar técnico
   - trabajo equipo
   - notas técnicas
   - text modal
7. **Utilities**
   - clamp, hidden, display helpers
8. **Animations**
   - unique keyframes only
9. **Mobile**
   - mobile header, drawer, bottom nav

## Phase 4 — Normalize & Simplify
- Replace inline style fragments in JS/HTML with class-based styles.
- Merge duplicate definitions (e.g., `.btn`, `.badge`).
- Remove unused color variants, keep only used variants.
- Ensure only one definition per animation name.

## Phase 5 — Verification
- Visual sweep: desktop + mobile view
- Check critical flows:
  - filters
  - acciones de orden
  - intervención
  - accesorios
  - modals
- Confirm no console errors

## Deliverables
- A cleaned, smaller `public/css/ordenes-index.css`
- A quick change log at top of file

## Notes
- **Aggressive** means we remove unused selectors quickly but in batches to avoid regressions.
- No split into multiple files (single-file requirement).
